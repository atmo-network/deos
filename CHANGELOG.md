## 0.6.4: Axial Router Contract Honesty and Recipient-Aware TMC Mint

- `[Axial Router / TMC] Recipient-aware mint delivery`: Threaded `recipient` through `TmcInterface::mint_with_distribution` and `pallet_tmc::mint_with_distribution` so the freshly minted user allocation lands on the router's requested `recipient` (collateral still leaves the minter; the zap allocation still lands in the protocol sink). Router DirectMint quote, route selection, slippage, return value, and `SwapExecuted.amount_out` now use recipient allocation rather than total curve emission. Impact: the mint route no longer silently ignores `recipient` or overstates user output, so user/system swaps through the TMC-mint path deliver and report output to the intended account; added recipient-delivery and recipient-slippage regression tests.
- `[Axial Router / TMC API] Mint-preview naming cleanup`: Renamed the TMC total-emission helper to `calculate_total_mint` and the router-facing TMC adapter quote to `calculate_recipient_receives`. Impact: API names now encode the total-emission vs recipient-output boundary, preventing future route-selection or slippage code from confusing protocol allocation with user output.
- `[Axial Router] Mechanism-only route selection`: Removed the vestigial `efficiency_score` (its `/1000` price-impact penalty and always-zero route fees made selection collapse to pure max-output anyway) and made route selection explicitly pure max-output. Impact: route selection now matches the documented `Mechanism-Over-Policy` contract; `price_impact` and `total_fees` remain on `RouteComparison`/`RouterQuote` as informational quote fields only, not selection inputs.
- `[Axial Router] Honest error surface`: Removed the dead `From<DispatchError>` impl (a landmine that collapsed any dispatch error into `FeeRoutingFailed`) and made direct-route price-deviation failures surface `PriceDeviationExceeded` instead of the misleading `NoRouteFound`. Impact: callers get the real failure reason and the error table matches runtime behavior.
- `[Axial Router] Honest test coverage`: Replaced the tautological `sandwich_attack_simulation` (which passed on any feeless router and proved no real sandwich resistance) with a `round_trip_buy_sell_is_net_negative_test` that characterizes round-trip execution cost, plus an explicit note that this launch line has no commit/reveal or ordering protection. Impact: test coverage no longer claims a MEV-resistance guarantee the code does not provide.
- `[Docs] Axial Router and TMC architecture sync`: Corrected the route-selection policy (max-output, not efficiency-score), the `PriceOracle` signature (no `tvl` arg), the multi-hop protection asymmetry (direct routes: EMA + slippage; multi-hop: slippage only), the `TrackedAssets` observability role, the `quote_price` vs `quote_exact_input` distinction, the mint-delivery recipient flow, and toned down the `eliminates Flash Loan/Dust attacks` overclaim into an honest mitigation statement. Impact: the architecture docs no longer overstate economic guarantees beyond what the shipped runtime enforces.
- `[Backlog] ZapManagerAccount rename debt`: Split the legacy `ZapManagerAccount` -> liquidity-actor-account rename (AGENTS.md §2 AAA Abstraction Lift) into its own bounded backlog slice rather than half-doing it inside this honesty pass. Impact: the rename epic is tracked as a complete one-pass slice instead of accumulating partial rename debt.
- `[Release] Rust workspace package markers`: Synchronized the template workspace Cargo package versions and lockfile entries to `0.6.4`. Impact: Rust artifacts now advertise the same release line as the changelog instead of carrying stale `0.5.x` / `0.6.2` package markers.
