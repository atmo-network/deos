# DEOS Framework Delivery History

> Canonical delivery history for the current DEOS repository line.
>
> This repository restarted its own release line at `0.0.0` after the move into the standalone DEOS monorepo. The changelog focuses on shipped outcomes and their impact, not intermediate refactor steps, package-version chores, or implementation diary entries.

## 0.7.3: Progress-Preserving AAA Continuation

- `AAA / Sparse progress preservation`: Added Mutable-only `RetryLater` over explicit Permanent/Temporary adapter failures, one scalar unresolved-step cursor, frozen typed suffix inputs, cumulative outcomes, and one logical-run nonce across attempts. Task-scoped rollback keeps earlier successful steps committed; retries start at the same cursor, while Permanent failure and ordinary one-attempt success write no Continuation. Impact: multi-step economic plans can survive temporary middle-step incapacity without replaying swaps, burns, or other committed prefixes and without becoming a general-purpose VM.
- `AAA / Canonical retry scheduling and economics`: Reused the existing monotonic paged FIFO and exact wakeup substrate for cooldown-driven retries, preserving one live ticket/wakeup and latching concurrent signals for the next logical run. Retry admission, Weight, User fees, spendability, and frozen `PercentageOfTrigger` snapshots cover only the unresolved suffix; funding arriving during suspension remains pending until full success. Impact: progress preservation adds no second scheduler, inbox, full-plan checkpoint, compensation path, or off-chain correctness dependency, while retry cost remains proportional to unresolved work.
- `AAA / Cancellation, observability, and portability`: Added Mutable-only call `23` for deterministic cancellation plus `CycleSuspended`, `CycleContinued`, and `CycleCancelled` event boundaries around one cumulative terminal summary. Plan/policy/schedule replacement, deactivation, terminal transition, and pure close share actor-local invalidation without compensation, prefix rollback, funding promotion, or sovereign-balance movement. The independent zero-topology runtime proves User/System continuation, all error policies, Executive ingress during suspension, disabled adapters, exact-output User swaps, System-only minting, metadata, try-state, and no-std operation. Impact: downstream runtimes and clients receive one bounded canonical current-run surface and can materialize unbounded attempt history from `(aaa_id, cycle_nonce, attempt)` event correlation.
- `AAA / Production evidence and client projection`: Generated optimized suspension, retry, completion, cancellation, and suffix-admission weights; synchronized the specification, architecture, embedding/read-model contracts, pallet/runtime entrypoints, PAPI metadata, reference-client authoring/status projection, and bilingual wiki. Generated AAA weights hash is `2c74e92a46727fa5ed359c9cdce14296885b22446be415fa0f0bd252ede18f38`; compressed production Wasm hash is `b858a8747ead30a5e2cda21f174d77f56ef7e9c1e063241682084fbe605717c9`. Impact: the `0.7.3` package line exposes one reviewable contract with production-bound RefTime/ProofSize evidence and no fake pre-launch migration ceremony.
- `AAA / Package ownership`: Moved the zero-topology embedding runtime beside the pallet source at `pallets/aaa/embedding-runtime` while preserving it as the separate non-published `pallet-aaa-embedding-fixture` Cargo consumer. Updated workspace and release-gate routing across default, DEX, try-runtime, benchmark, and no-std profiles. Impact: repository topology now identifies `pallet-aaa` as the reusable package owner without weakening independent downstream-consumer evidence.

- `AAA / Control-plane contract`: Defined chain- and metadata-bound canonical plan artifacts around exact `ProgramInput` SCALE bytes, deterministic identity, lossless typed projection, ordered diff, forecast/simulation provenance, governance payload composition, and materialized history semantics. Impact: off-chain tooling has one falsifiable contract without turning JSON, cached projections, simulations, or indexer history into consensus truth.

Cross-genesis or cross-metadata artifacts fail as `IncompatibleUntilRebound`; focused generated-metadata fixtures cover Dormant and nested Active plans. Impact: authoring and review tools can share one tested artifact identity without handwritten runtime type indexes or implicit signing.
- `AAA / Static forecasting`: Mirrored pallet amount-resolution rules for fee reserve, minimum balance, trigger snapshots, last funding, minting, rounding, and staking shares over explicit observations. Added state/model pins plus separately aggregated RefTime, ProofSize, evaluation fees, execution upper fees, lifecycle overhead, and stale-head detection under the honest `StaticAllStepsReached` scope. Impact: runtime admission now binds to weights hash `2c74e92a46727fa5ed359c9cdce14296885b22446be415fa0f0bd252ede18f38` and Wasm hash `b858a8747ead30a5e2cda21f174d77f56ef7e9c1e063241682084fbe605717c9`; runtime-call parity remains a separate evidence gate.

## 0.7.1: AAA Semantic Hardening

Impact: unavailable capacity, governance pauses, and atomicity failures cannot masquerade as adapter faults, emit false closure, strand live actors, or expose policy knobs outside the bounded kernel.
- `Runtime / AAA production capacity`: Split block capacity 50/50 between dispatch and `on_idle`, intentionally removed the unused Operational reserve, and separated TMC collateral from minted-liquidity routing so BLDR collateral reaches its Liquidity Actor without a redundant Splitter hop. Optional compatibility-ingress drain, heavyweight wakeup-retry, and sweep-time terminal-close units remain metered durable carry-over rather than part of the same-block plan guarantee. Mixed Noop/transfer, maximum address-ingress plus XCM, and simultaneous wakeup-retry plus expired-actor close stress under that exact reserve prove durable FIFO carry-over, eventual queue convergence, retained cross-consensus triggers, close-tail liveness, no starvation, and no failed steps. Bucket unwind now uses two independently admitted single-step cycles — Bucket LP transfer followed by Treasury-owned liquidity removal of one configured system LP asset — so both underlying assets land directly in Treasury custody without the rejected dual-transfer tail; Bucket/Treasury pairing records topology rather than imposing sender-restricted ingress. Impact: the reference actor graph has an executable production envelope without sacrificing half-block transaction capacity, carrying unused topology, or confusing syntactic bounds with schedulable plans.
